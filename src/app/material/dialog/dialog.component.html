Dialog - Default MdDialog is a service, which opens dialogs components in the view.

This example demonstrates a true dialog service (similar to what was available in Material 1 - but not yet available out of the 'box' in Material 2). In a single component, we can define different types of dialogs. Also provides the ability to use an inline dialog using <ng-template>.

Features:
  • Sharing data with the Dialog component
  • Receive notifications from the dialog after it has been closed
  • Pass in external content
  • Inline using <ng-template>
  • Different flavors: Alert, Confirm, Save etc...

We can have custom backdrop classes in the config. Here the default is being applied. Try adding: cdk-overlay-light-backdrop for a different variation (this class is contained in the _dialog.scss partial). For no backdrop - add: none.

Sets hasBackdrop as a boolean. If has backdrop is set to false - content behind modal is accessible. To achieve a display of 'no backdrop' apply the custom backDrop class="none". This will maintain a higher z-index for dialog - while not displaying any backdrop whatsoever.
