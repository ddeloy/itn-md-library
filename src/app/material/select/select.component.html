Select Single Selection

MdSelectModule - Official Documentation

<md-select> is a form control for selecting a value from a set of options, similar to the native <select> element.

UI design and behaviour wise, it is exactly similar to input and textarea elements.

Usage

Select box is created by using md-select element. For each option you'd like in your select, add an md-option tag. Note that you can disable items by adding the disabled boolean attribute or binding to it.

Static Label and Placeholder

Similar to input and textarea elements, md-select elements have been modified to have a static label, and a non-floating placeholder which disappears as soon as the user selects a value. To achieve this, <md-select> is wrapped inside <label> element. Floating placeholder has been disabled by setting floatPlaceholder attribute of md-select to never.

Floating Placeholder

One can, however, still use floating placeholder with select boxes by either skipping the floatPlaceholder attribute completely on md-select or by setting it to auto.

Note: Do NOT wrap md-select inside a label element when using floating placeholder.

Select Box with Floating Placeholder 
<md-select placeholder="Favorite Drink" [(ngModel)]="favDrink"
  floatPlaceholder="auto" #favDrinkControl="mdSelect">
  <md-option *ngFor="let drink of drinks" [value]="drink.value" [disabled]="drink.disabled">
    {{'{'}}{{'{'}} drink.viewValue {{'}'}}{{'}'}}
  </md-option>

Underline Color and Border

Similar to input and textarea elements, md-select elements have been modified to have border bottom on mobile screens and border on all four sides on tabs and bigger screens. These borders change color on focus, and this color can be set using dividerColor attribute on md-select. Supported values for this attribute are primary, accent and warn. If none of these value are present or if this attribute is missing, then a default color will be used.

Select with dividerColor set 
<label>
  <span>Primary</span>
  <md-select placeholder="Drink" dividerColor="primary" [(ngModel)]="currentDrink1"
    floatPlaceholder="never" #drinkControl1="mdSelect">
    <!-- set dividerColor to primary or accent or warn -->
    <md-option *ngFor="let drink of drinks" [value]="drink.value" [disabled]="drink.disabled">
      {{'{'}}{{'{'}} drink.viewValue {{'}'}}{{'}'}}
    </md-option>
  </md-select>
</label>

Compact Select Boxes

Select Boxes can be "compactified" - having lesser height and padding - by adding compact to md-select.

NOTE: compact class does NOT have any effect on elements in mobile view.

Multiple Selection Variation - Popover

This variation by default shows only the labels of the selected options. To make/change the selection, a triggering element, like a button or a span, is clicked. The user is then presented with a list of options along with checkboxes to select/de-select. Also available is a search input to search through the available options.

This variation is created by using a custom SelectPopoverComponent, with selector as select-popover.

Inputs to select-popover:

  • label: label/text to be shown at the top of the component.
  • options: list of options as an array. Please see below code sample for structure.

Outputs/Events of select-popover:

  • onClose: emitted when the component is closed. Returns the updated options array.

Custom popover-container and popover classes have been added to simulate behaviour of a popover element. Element with popover class will be hidden by default, and will fade in only when shown class is added to it. Element with popover class should be placed inside an element with popover-container class.

Hint Label and Error Message

Hint label is an indicative text that appears below the select box. These provide additional information about the selected value to the user, as to what is expected or what the user should keep in mind while selecting a value.

Hint labels for md-select elements are set by using an md-hint element inside the md-select, which takes an align attribute containing the side as start or end.

Error messages appear below the select box. These are hidden by default and will appear on an invalid input after the user has interacted with the element. For example, if the md-select element is marked required but the user doesn't select a value or selects a None value.

Error messages can be set by using md-hint elements having error class inside the md-select, with hidden property bound to desired property of the element. For example, [hidden]="name.valid || name.pristine".
