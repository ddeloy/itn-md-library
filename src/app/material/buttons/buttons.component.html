Buttons md-button is an HTML button or a tag enhanced with styling and animation to match the Material Design button spec

MdButtonModule - Official Documentation

<button> and <a> elements are always used in order to provide the most straightforward and accessible experience for users. A <button> element should be used whenever some action is performed. An <a> element should be used whenever the user will navigate to another view.

There are five basic button variants, each applied as an attribute:

Attribute Description Example
md-button Rectangular button w/ no elevation. Also known as Flat Buttons
md-raised-button Rectangular button w/ elevation Also known as Default Buttons
md-icon-button Circular button with a transparent background, meant to contain an icon Also known as Icon Buttons
md-fab Circular button w/ elevation, defaults to theme's accent color
md-mini-fab Same as md-fab but smaller

Additional variants available:

Attribute Description Example
md-round-button Rectangular button with rounded corners. Also known as Round Buttons
md-button-sm For smaller version of the buttons Also known as Small Buttons

Usage

Required button is created using button tag along with above mentioned attributes. These attributes can be used in combination to create desired variant. For example, md-raised-button md-round-button are used together to create a round raised/default button.

Default Buttons

These are the buttons which shall be used by default for all the user actions, unless it is mentioned to use some other variant of the button element.

Default buttons, also known as Raised Buttons, are created using md-raised-button attribute on button element.

Default Button 
<button md-raised-button color="primary">Click me!</button>
Default Button - Disabled 
<button md-raised-button disabled color="primary">Disabled</button>

Flat Buttons

These buttons do not have any background and elevation, and shall be used for non-primary actions.

Flat buttons are created using md-button attribute on button element.

Flat Button 
<button md-button color="primary">Flat Button</button>
Flat Button - Disabled 
<button md-button disabled color="primary">Flat Button</button>

Round Buttons

This variant adds rounded-corners to the default buttons.

Round buttons are created using md-round-button attribute along with md-raised-button on button element.

Round Button 
<button md-raised-button md-round-button color="primary">Click me!</button>

Icon Buttons

This is a circular button with an icon denoting the action and no text.

Icon buttons are created adding md-icon-button attribute along with md-raised-button or md-button attribute on button element. Icon to be shown is mentioned by using md-icon element inside the md-button element. Md-icons use 'Material Icons' font-family to create the icon, and therefore, inner text of these elements should be one of the keywords from the list of Material Icons.

Icon Button (Default Version) 
<button md-raised-button md-icon-button color="primary"><md-icon>search</md-icon></button>
Icon Button (Flat Version) 
<button md-button md-icon-button color="primary"><md-icon>info_outline</md-icon></button>

Small Buttons

This variant creates a small or more compact version of the above mentioned buttons.

Small buttons are created adding md-button-sm attribute along with above mentioned attributes on button element.

Small Default Button 
<button md-raised-button md-button-sm color="primary">Small Button</button>
Small Flat Button 
<button md-button md-button-sm color="primary">Small Button</button>
Small Icon Button 
<button md-raised-button md-icon-button md-button-sm color="primary"><md-icon>search</md-icon></button>

Theming

Buttons can be colored in terms of the current theme using the color property to set the background color to primary, accent, or warn. By default, only FABs are colored; the default (when no color is mentioned) background color for md-button and md-raised-button matches the theme's background color.

Default Button - Primary 
<button md-raised-button color="primary">Primary</button>
Default Button - Accent/Secondary 
<button md-raised-button color="accent">Accent/Secondary</button>

Dropdown and Split Buttons

Dropdown buttons toggle a contextual, dropdown menu displaying a list of actions/links. Button is created using one of the above variants, and corresponding dropdown menu is created using md-menu element. Button and menu are tied up using [mdMenuTriggerFor] property binding which provided the value equal to id of the md-menu element. Button and menu are wrapped inside div with class btn-group.

A variant of dropdown buttons, known as Split buttons, allows for a single or more button(s) performing the most primary/common action(s) along with a small icon button which acts as trigger for toggling the dropdown menu to show secondary actions.

Default Buttons are available in both - Default/Raised and Flat - variants.

Dropdown Button - Single Button 
<div class="btn-group">
  <button [mdMenuTriggerFor]="appMenuEx1" md-raised-button color="primary">
    Actions
    <md-icon>keyboard_arrow_down</md-icon>
  </button>
  <md-menu #appMenuEx1 [overlapTrigger]="false">
  <!-- [overlapTrigger]="false" makes dropdown menu to not cover the button -->
    <li md-menu-item>First menu item</li>
    <li md-menu-item>Second menu item</li>
    <li md-menu-item>Third menu item</li>
    <li class="divider"></li>
    <li md-menu-item>Fourth menu item</li>
  </md-menu>
</div>
