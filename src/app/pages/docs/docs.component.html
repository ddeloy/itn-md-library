MdButtonModule Official Documentation

Angular Material buttons are native <button> or <a> elements enhanced with Material Design styling and ink ripples.

Native <button> and <a> elements are always used in order to provide the most straightforward and accessible experience for users. A <button> element should be used whenever some action is performed. An <a> element should be used whenever the user will navigate to another view.

There are five basic button variants, each applied as an attribute:

Attribute Description Example
md-button Rectangular button w/ no elevation. Also known as Flat Buttons
md-raised-button Rectangular button w/ elevation Also known as Default Buttons
md-icon-button Circular button with a transparent background, meant to contain an icon Also known as Icon Buttons
md-fab Circular button w/ elevation, defaults to theme's accent color
md-mini-fab Same as md-fab but smaller

Additional variants available:

Attribute Description Example
md-round-button Rectangular button with rounded corners. Also known as Round Buttons
md-button-sm For smaller version of the buttons Also known as Small Buttons

Dropdown and Split Buttons

Dropdown buttons toggle a contextual, dropdown menu displaying a list of actions/links. Button is created using one of the above variants, and corresponding dropdown menu is created using md-menu element. Button and menu are tied up using [mdMenuTriggerFor] property binding which provided the value equal to id of the md-menu element. Button and menu are wrapped inside div with class btn-group.

A variant of dropdown buttons, known as Split buttons, allows for a single or more button(s) performing the most primary/common action(s) along with a small icon button which acts as trigger for toggling the dropdown menu to show secondary actions.

Usage

Required button is created using button tag along with above mentioned attributes. These attributes can be used in combination to create desired variant. For example, md-raised-button md-round-button are used together to create a round raised/default button.

Theming

Buttons can be colored in terms of the current theme using the color property to set the background color to primary, accent, or warn. By default, only FABs are colored; the default background color for md-button and md-raised-button matches the theme's background color.

Examples

Default/Raised Button (with Primary color) 
<button md-raised-button color="primary">Click me!</button>
Default/Raised Round Button (with Accent/Secondary color) 
<button md-raised-button md-round-button color="accent">Click me!</button>
Default/Raised Button - Disabled 
<button md-raised-button disabled color="primary">Disabled</button>
Small Flat Button 
<button md-button md-button-sm color="warn">Small</button>
Icon Button 
<button md-raised-button md-icon-button color="primary"><md-icon>search</md-icon></button>

    MdInputModule Official Documentation

    <md-input-container> is a wrapper for native input and textarea elements. This container applies Material Design styles and behavior while still allowing direct access to the underlying native element.

    The native input or textarea element wrapped by the md-input-container must be marked with the mdInput directive.

    Label and Placeholder

    By default, in Material design, input and textarea elements have floating placeholder, which when text is present in those elements, floats above the input zone.

    However, these elements have been modified to have a static label, and a non-floating placeholder which disappears as soon as the user starts typing in the input zone. To achieve this, <md-input-container> is wrapped inside <label>. Floating placeholder has been disabled by setting floatPlaceholder attribute of md-input-container to never.

    Underline Color and Border

    By default, in Material design, input and textarea elements have a line under them that changes color whenever these elements gain focus.

    However, these elements have been modified to have border bottom on mobile screens and border on all four sides on tabs and bigger screens. These borders change color on input focus, and this color can be set using dividerColor attribute on md-input-container. Supported values for this attribute are primary, accent and warn. If none of these value are present or if this attribute is missing, then a default color will be used.

    Prefix and Suffix

    HTML can be included before, and after the input or textarea element tag, as prefix or suffix. It will be included inside the border of these elements, and clicking it will focus the input.

    Adding the mdPrefix attribute to an element inside the md-input-container will designate it as the prefix. Similarly, adding mdSuffix will designate it as the suffix.

    Hint Label and Error Message

    Hint label is an indicative text that appears below the input and textarea elements. These provide additional information about the input to the user, as to what is expected or what the user should keep in mind while entering the value. Hint labels can be set using hintLabel attribute on md-input-container.

    Error messages appear beneath the input and textarea elements. These are hidden by default and will appear on invalid inputs after the user has interacted with the element. For example, if the input element is marked required but the user deletes the entered value. Error messages can be set by using div elements having error class inside the md-input-container, with hidden property bound to desired property of the element. For example, [hidden]="name.valid || name.pristine". Hint labels and error messages can both be set on an element. However, hint labels will be hidden whenever an error message is displayed.

    Examples

    Default and Read-only Inputs 
    <label>
  <span>Name</span>
  <md-input-container floatPlaceholder="never">
    <input mdInput placeholder="John Smith" type="text">
  </md-input-container>
</label>
<!-- Read-only input -->
<label>
  <span>Read-only</span>
  <md-input-container floatPlaceholder="never">
    <input mdInput placeholder="enter a value" type="text" value="This is readonly" readonly>
  </md-input-container>
</label>
    Inputs with dividerColor set 
    <label>
  <span>Primary Color</span>
  <md-input-container dividerColor="primary" floatPlaceholder="never">
  <!-- set dividerColor to primary or accent or warn -->
    <input mdInput placeholder="enter a value" type="text">
  </md-input-container>
</label>
    Input with Prefix and Suffix 
    <label>
  <span>With Prefix and Suffix</span>
  <md-input-container floatPlaceholder="never">
    <span mdPrefix>+1</span>
    <input mdInput placeholder="Telephone" type="tel">
    <span mdSuffix>mode_edit</span>
  </md-input-container>
</label>
    Input with Hint Label 
    <label>
  <span>With Hint Label</span>
  <md-input-container hintLabel="Should be easy to remember but difficult to guess" floatPlaceholder="never">
    <input mdInput placeholder="Password" type="password">
  </md-input-container>
</label>
    Input with Error Message 
    <label>
  <span>With Error Message</span>
  <md-input-container hintLabel="Should be easy to remember" dividerColor="primary" floatPlaceholder="never">
    <input mdInput placeholder="John Smith" type="text" [(ngModel)]="model.name"
      #name="ngModel" required>
    <!-- do NOT forget to set ngModel on this element -->
    <!-- corresponding model object is set in TS file -->
    <!-- model = {{ '{' }}'id': 18, 'name': 'Dr IQ'{{ '}' }}; -->
    <div [hidden]="name.valid || name.pristine" class="error">Name is required</div>
  </md-input-container>
</label>
         MdSelectModule Official Documentation

    <md-select> is a form control for selecting a value from a set of options, similar to the native <select> element.

    UI design and behaviour wise, it is exactly similar to input and textarea elements.

    Usage

    Select box is created by using md-select element. For each option you'd like in your select, add an md-option tag. Note that you can disable items by adding the disabled boolean attribute or binding to it.

    Label and Placeholder

    Similar to input and textarea elements, md-select elements have been modified to have a static label, and a non-floating placeholder which disappears as soon as the user selects a value. To achieve this, <md-select> is wrapped inside <label>. Floating placeholder has been disabled by setting floatPlaceholder attribute of md-select to never.

    Underline Color and Border

    Similar to input and textarea elements, md-select elements have been modified to have border bottom on mobile screens and border on all four sides on tabs and bigger screens. These borders change color on focus, and this color can be set using dividerColor attribute on md-select. Supported values for this attribute are primary, accent and warn. If none of these value are present or if this attribute is missing, then a default color will be used.

    Hint Label and Error Message

    Hint label is an indicative text that appears below the select box. These provide additional information about the selected value to the user, as to what is expected or what the user should keep in mind while selecting a value.

    Hint labels for md-select elements are set by using an md-hint element inside the md-select, which takes an align attribute containing the side as start or end.

    Error messages appear below the select box. These are hidden by default and will appear on an invalid input after the user has interacted with the element. For example, if the md-select element is marked required but the user doesn't select a value or selects a None value.

    Error messages can be set by using md-hint elements having error class inside the md-select, with hidden property bound to desired property of the element. For example, [hidden]="name.valid || name.pristine".

    Examples

    Default and Disabled Select Boxes 
    <label>
  <span>Default</span>
  <md-select placeholder="Drink" [(ngModel)]="currentDrink"
    floatPlaceholder="never" #drinkControl="ngModel">
    <md-option *ngFor="let drink of drinks" [value]="drink.value" [disabled]="drink.disabled">
      {{'{'}}{{'{'}} drink.viewValue {{'}'}}{{'}'}}
    </md-option>
    <!-- corresponding drinks object is set in TS file -->
    <!-- drinks = [ -->
    <!--  {{ '{' }}value: null, viewValue: 'None'{{ '}' }}, -->
    <!--  {{ '{' }}value: 'coke-0', viewValue: 'Coke'{{ '}' }}, -->
    <!--  {{ '{' }}value: 'water-1', viewValue: 'Water'{{ '}' }}, -->
    <!--  {{ '{' }}value: 'pepper-2', viewValue: 'Dr. Pepper', disabled: true{{ '}' }}, -->
    <!--  {{ '{' }}value: 'coffee-3', viewValue: 'Coffee'{{ '}' }}, -->
    <!--  {{ '{' }}value: 'tea-4', viewValue: 'Tea'{{ '}' }}, -->
    <!-- ]; -->
  </md-select>
</label>

    This is required
  </md-hint>
